Architecture Deep-Dive

How the Server Works

A component-by-component walkthrough of the request lifecycle, thread model, and key design decisions.

Request Lifecycle

1

accept()

The main thread blocks on accept(). When a client connects, a file descriptor and IP address are captured.

→
2

ThreadPool::submit()

The fd is wrapped in a lambda and pushed into the FIFO task queue. A sleeping worker is woken via notify_one().

→
3

parse_request()

The worker reads the socket incrementally using select() + recv(), assembling the request until \r\n\r\n is found.

→
4

Router::route()

URI is decoded, sanitised, and resolved to a filesystem path. File is read, MIME type detected, and a 200/403/404/500 response is built.

→
5

send() + close()

Headers then body are sent in 64 KiB chunks via send(MSG_NOSIGNAL). The socket is closed and metrics are updated atomically.

Key Design Decisions

🔄 Fixed Pool vs. Thread-per-Request

Thread creation (≈ 50–200 µs on Linux) would dominate latency for short requests. A fixed pool amortises that cost to zero and bounds memory usage regardless of connection rate.

📋 Queue Swap in Logger

The I/O thread swaps the shared queue with a local empty queue under the mutex, then releases the lock before writing to disk. Workers contend only for a pointer swap, not a file write.

⚛️ Atomic Stats Counters

Performance counters (requests, latency, concurrency) use std::atomic instead of a mutex. This avoids false sharing and allows lock-free updates from all worker threads simultaneously.

🛡️ Double Traversal Guard

Path sanitisation happens twice: in the parser (lexically) and in the router (after weakly_canonical() resolution). Both layers must agree the path is under document_root.

🔌 SO_REUSEADDR

The listening socket sets SO_REUSEADDR so that after a crash or restart, the port is available immediately instead of waiting up to 4 minutes for TIME_WAIT to expire.

📡 SIGPIPE Ignored

Writing to a socket whose remote end has closed raises SIGPIPE, which by default kills the process. We ignore it globally and instead detect write errors via the return value of send().
