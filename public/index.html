Core Features

🧵

Fixed Thread Pool

Worker threads are created once at startup. Requests are dispatched via a lock-free-style FIFO queue guarded by a condition variable – zero thread creation overhead per request.

🔌

POSIX Sockets

Raw socket() → bind() → listen() → accept() pipeline. SO_REUSEADDR enables instant restart. EINTR-safe accept loop survives signals gracefully.

📨

HTTP/1.1 Parser

Incremental socket reader with select()-based timeout. Parses method, URI, version, and all headers. URL-decodes paths and sanitises against directory traversal.

🛡️

Synchronisation

RAII locking (std::lock_guard, unique_lock), condition variables, and atomic counters eliminate data races. Verified with AddressSanitizer + ThreadSanitizer.

📁

Static File Server

Serves HTML, CSS, JS, PNG, JPG, PDF and more. Automatic MIME detection, proper Content-Length, and chunked 64 KiB sends for large files.

🔒

Security

Double directory-traversal guard using std::filesystem canonicalisation. SIGPIPE ignored to prevent crashes. Max header size enforced to resist memory exhaustion.

📊

Metrics

Atomic counters track total requests, average latency, peak concurrent connections, and requests-per-second. Printed on clean shutdown.

📝

Async Logger

Dedicated I/O thread drains a queue with the swap trick – worker threads never block on disk writes. Timestamps at millisecond resolution.

Quick Start

bash 
# Build
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)

# Run
./HTTPServer               # uses config.json
./HTTPServer myconfig.json # custom config

# Test
curl http://localhost:8080/
curl -I http://localhost:8080/styles.css
curl http://localhost:8080/nonexistent
